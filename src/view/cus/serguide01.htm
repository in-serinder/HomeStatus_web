多组关键词

搜寻方法用六组词语, 以"并/或 运算符" (AND/OR operator)连接。运算符依据通用法则，先做并连，再做或连。

因此，六组词语分成两类：第一类是第一行的三组词语，用或连。第二类是第二行的三组词语，用并连。两行再用"或"连接。

如果输入英文等字母文字，大写小写视作相同。

以下例子说明各种搜寻方法。

举例 1: 或连 - 任一词，不计顺序。(OR Operation - Any words, any order)

搜寻有关 “天堂”，“地狱”，“乐园” 的章节。这三个词语不必在同一章节出现。我们要含有任一词的所有章节，且不计词语顺序。

这类搜寻用或连。要输入“天堂”到第一组，“地狱”到第二组，“乐园”到第三组。全在第一行。

举例 2: 并连 - 全部词，不计顺序。(AND Operation - All words, any order)

搜寻有关 “神” 和 “国” 的章节。这两个词语要在同一章节出现。排除只出现 “神”，或只出现 “国”的章节。

这类搜寻用并连。要输入“神”到第四组，“国”到第五组。全在第二行。

举例 3: "并/或"齐用 (AND/OR Comination)

搜寻有关全部上述的观念。

这类搜寻"并/或"齐用。输入“天堂”，“地狱”，“乐园”到第一行。输入“神”，“国”到第二行。

举例 4: 单一确切词，无添加。(Exact words, no extension)

中文全部是单一确切词。 但如果输入英文等字母文字，例如"last"字，又不想搜出"everlasting", 可以在字的前后加上空白，输入" last "。中文就无此问题。

举例 5: 确切词，确切顺序。(Exact words, exact order)

要搜寻完整字句，例如"在前的将要在后"，要将整句输入任一组内，不能分散几组。

举例 6: 简单正则表达式 - 搜出任何字。(Regular Expression - Match any characters)

要搜寻 "前" "后" 两字，中间夹杂任何词句。模式是 "前.*后" (中间无空白)。 这是因为符号"."代表任何字。符号"*"代表之前符号的任何字数。例如，章节内有"在前的将要在后"一句，含"前"字，跟着是任何字任何字数，再跟着是"后"字，就会搜寻出来。

这方法效果和在第四组放"前"字，第五组放"后"字不同。那会将"在后的将要在前"也一同搜出来。因为 ".*" 计顺序, 而并连 (AND Operation) 是不计顺序的。

更高级搜寻:

六组关键词的每一组，都接纳 正则表达式 Regular Expression 的模式。

用适当模式搜寻，传道人和圣经学者，就能在几秒钟内，将切当资料找寻出来。
